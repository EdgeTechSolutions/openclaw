## OneDrive / SharePoint Issues
- Luka had trouble copying files from a SharePoint Shared Library synced via OneDrive on macOS
- Files appeared local but were behind OneDrive's File Provider extension — all file operations went through OneDrive process causing timeouts
- `mv` and `cp` both timed out; `rsync` partially worked but some files (e.g. `.claude/settings.local.json`) still timed out on `mmap`
- Solution: cloned from GitHub instead; OneDrive shared library was in broken sync state
- **Lesson**: OneDrive Shared Libraries on macOS often show as virtual mounts without proper Files On-Demand support; SharePoint web download or git clone are more reliable

## Quick Answers Given
- `git config core.fileMode false` — ignore permission-only changes
- Revert permission-only changes: filter `git diff` for "mode change" lines
- `ollama list | tail -n +2 | awk '{print $1}' | xargs -I {} ollama rm {}` — remove all models
- Ollama model storage on Windows: `C:\Users\<username>\.ollama\models\`
- Brew checksum error on gradle: `rm -rf $(brew --cache)/downloads/*gradle*` then retry
- OpenCode (Tauri) eating 4.35 GB RAM + 1.54 GB CLI = ~5.9 GB → likely memory leak; 13.68 GB swap causing freezes
